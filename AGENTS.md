# Gold Summit Capital Project Instructions

当 Codex 或其他 AI 助手进入本文件夹时，请先阅读以下文件，并按顺序理解项目上下文：

1. `00_START_HERE.md`
2. `01_PROJECT_STATUS.md`
3. `02_NEXT_ACTIONS.md`
4. `03_PRODUCT_BLUEPRINT.md`
5. `04_ROADMAP.md`

工作原则：

- 把 Gold Summit Capital 当作一个持续迭代的创业项目，而不是一次性文档。
- 每次对话后，如果产生新决策、新方案或新任务，需要更新 `01_PROJECT_STATUS.md`、`02_NEXT_ACTIONS.md` 或 `07_DECISION_LOG.md`。
- 所有建议都应围绕两个核心产品：基金管理服务，以及客户专属的投资、教育与传承内容中枢。
- 不要默认增加复杂功能。优先验证客户是否感知到价值，再考虑平台化、自动化和社区化。
- 对基金绩效、持仓、客户资料、活动记录等敏感信息，默认采用保守披露原则，并提醒需要合规或法律确认。
