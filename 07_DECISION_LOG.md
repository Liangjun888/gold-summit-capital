# 决策记录

## 2026-05-19

### 决策 1：网站分为公共区域和客户专属登录区

公共区域用于公司介绍、团队简介和理念表达。

客户专属登录区是更重要的产品载体，用于交付月报、研究、基金绩效、活动记录和教育传承内容。

### 决策 2：暂不做“创始人的餐桌”

原因：

客户与创始人已有长期关系，平时会通电话或私下吃饭，不需要在网站中包装成独立栏目。

### 决策 3：暂不做正式客户访谈机制

原因：

创始人与客户及其子女都比较熟悉，不需要额外设计访谈流程。

### 决策 4：活动记录采用结构化内容，而不是照片墙

原因：

活动现场不适合拍摄客户正脸或披露身份，但仍需沉淀公司做过的事情。采用 Event Capsule 活动胶囊格式，记录主题、洞察、匿名化 Q&A、推荐阅读和家庭讨论问题。

### 决策 5：MVP 优先做低复杂度版本

第一版重点是可交付、可维护、可展示。

暂不做社区、论坛、复杂 App、客户聊天和大型课程平台。

### 决策 6：网站 V1 优先公共形象展示

第一版优先完成公共网站，而不是直接开发完整客户后台。

公共网站承担第一印象：让被介绍来的潜在客户快速判断 Gold Summit Capital 是否专业、稳重、有温度、值得进一步沟通。

客户专属区作为 V1.5 预留。

### 决策 7：公共网站采用保守披露原则

公共网站不公开具体历史业绩、收益目标、基金条款、起投金额、管理费、业绩分成、锁定期、申赎细则、具体持仓和未授权顾问网络名单。

月报、研究、基金信息和活动记录进入客户专属区或受控材料。

### 决策 8：公共网站 V1 页面结构

V1 采用轻量多页面官网，不做一页式长页。

页面包括：

- 首页
- 关于我们
- 投资与传承理念
- 团队
- 客户登录

### 决策 9：客户登录页 V1 只做私密入口说明

V1 不做真实登录，不做假登录表单，不开放注册。

客户登录页采用“私密入口 + 未来内容中枢”的口径，说明该区域未来将承载月报、研究、活动记录与传承内容，并仅向受邀客户开放。

### 决策 10：视觉方向参考 J.P. Morgan Private Bank

视觉方向参考 J.P. Morgan Private Bank 的私人银行式克制审美，但不复制品牌资产或具体版式。

Gold Summit Capital 采用深棕、金棕、米白、暖灰、大留白、高级衬线标题和清晰无衬线正文，整体呈现现代精品投资机构气质。

### 决策 11：V1 使用文字型品牌标识

V1 先使用 `Gold Summit Capital` 文字型 wordmark，不让复杂 Logo 设计阻塞网站进度。

### 决策 12：V1 不做公开 Insights 栏目和开放联系表单

V1 不做公开洞察栏目，避免内容维护压力和公开披露风险。

V1 不做开放联系表单，仅保留通过介绍人或 Gold Summit Capital 团队联系的私密说明。

### 决策 13：V1 使用语言路径

V1 只上线简体中文，但 URL 从第一版开始采用语言路径：

- `/zh-CN/`
- `/zh-CN/about`
- `/zh-CN/approach`
- `/zh-CN/team`
- `/zh-CN/client-portal`

未来预留 `/zh-HK/` 和 `/en/`。

Gold Summit Capital V1 视觉采用深棕、金棕、米白、暖灰、石墨灰和少量低饱和绿色灰作为平衡色。主风格是现代精品投资机构与金融出版感，避免奢华化、销售化、过度卡片化和单一棕色主导。

### 决策 17：技术与部署方案推荐 Astro + Netlify

技术与部署方案比较文件为：

- `14_TECH_DEPLOYMENT_OPTIONS.md`

推荐公共网站 V1 使用 Astro 静态网站 + Netlify Git-based deploy。

理由：

- V1 是内容型公共官网，Astro 更适合快速、轻量、SEO 友好的内容网站。
- 从第一版开始保留 `/zh-CN/`、未来 `/zh-HK/` 和 `/en/` 的多语言结构。
- Netlify 适合静态网站部署、预览链接、自定义域名、SSL 和后续 Functions / Identity 扩展。
- V1 不做真实客户登录，V1.5 再单独评估认证、资料库、权限和访问日志。

备选方案包括 Astro + Cloudflare Pages、Next.js + Vercel、纯静态 HTML/CSS 和 Webflow/Wix/Squarespace，但当前均不作为首选。

### 决策 18：先完成零依赖可运行网站原型

当前本地环境没有可用 `npm`，系统默认 `node` 也被拒绝访问。为避免技术环境阻塞公共形象展示，网站 V1 先采用零依赖静态生成器完成可运行原型。

该决定不改变长期推荐方向。Astro + Netlify 仍是正式生产技术路线的首选，等具备 `npm`、GitHub 仓库和 Netlify 项目后再决定是否迁移。

### 决策 23：团队页从 V1 移除

创始人及合伙人不进行实名、照片与履历披露。在无实名内容支撑的情况下，独立团队页价值不足，整体从 V1 移除。

处理方式：

- 导航与 URL 中删除 `/zh-CN/team/`，路由保留为未来预留项
- 团队可信度功能由「关于我们」页的匿名能力矩阵承接（跨市场配置、亚洲金融市场经验、研究与产品能力、产业与一级市场资源、运营与服务商协同）
- 全站不使用任何人物照片
- 文案中"创始人"仅以职能身份出现，不出现姓名
